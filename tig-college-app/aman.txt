1. Onboard New Staff
Creates a single new staff member in the system.

Endpoint: POST /api/v1/staff

Request Body:

A JSON object with the staff member's details. FirstName, CollegeMasterID, and DepartmentMasterID are the minimum required fields.

JSON

{
  "FirstName": "John",
  "LastName": "Doe",
  "PersonalEmail": "john.doe@example.com",
  "MobileNumber": "9876543210",
  "DateOfBirth": "1990-06-15",
  "Designation": "Assistant Professor",
  "CollegeMasterID": 1,
  "DepartmentMasterID": 24,
  "DateOfJoining": "2024-08-01"
}
Expected Output (201 Created):

A JSON object representing the newly created staff profile.

JSON

{
    "FacultyMasterId": 123,
    "EID": "some-new-eid",
    "Name": "John Doe",
    "MobileNumber": "9876543210",
    "PersonalEmail": "john.doe@example.com",
    "Designation": "Assistant Professor",
    "CollegeMasterId": 1,
    "StreamMasterId": 24,
    "ProfessionalEmail": "john.d@institution.ac.in",

    "StreamName": "Computer Science",

4. Search for Staff
Finds staff members based on flexible search criteria provided as query parameters.

Endpoint: GET /api/v1/staff/search

Query Parameters: At least one is required.

id: .../search?id=6

name: .../search?name=John

email: .../search?email=john.doe@example.com

phone: .../search?phone=9876543210

Request Body: None.

Expected Output (200 OK):

An array of staff profile objects that match the search criteria.

JSON

[
  {
    "FacultyMasterId": 123,
    "EID": "some-new-eid",
    "Name": "John Doe",
    "MobileNumber": "9876543210",
    "PersonalEmail": "john.doe@example.com",
    "...": "..."
  }
]
5. Update Staff Profile
Updates the details of an existing staff member.

Endpoint: PUT /api/v1/staff/{staffId} (e.g., /api/v1/staff/6)

Request Body:

A JSON object containing only the fields you want to update.

JSON

{
  "MobileNumber": "9999988888",
  "PersonalEmail": "john.doe.new@example.com",
  "Designation": "Professor"
}
Expected Output (200 OK):

A confirmation message that the update is pending approval.

JSON

{
  "message": "Update request submitted for approval."
}
6. Deactivate (Soft Delete) Staff
Marks a staff member as inactive in the system.

Endpoint: DELETE /api/v1/staff/{staffId} (e.g., /api/v1/staff/6)

Request Body: None.

Expected Output (200 OK):

A success confirmation message.

JSON

{
  "message": "Staff member deactivated successfully."
}
7. Reactivate Staff
Reactivates an inactive staff member.

Endpoint: POST /api/v1/staff/{staffId}/reactivate (e.g., /api/v1/staff/6/reactivate)

Request Body: None.

Expected Output (200 OK):

The full profile object of the reactivated staff member.

JSON

{

    "EID": "fjCHqIdJTJirplNd",
    "Name": "Dr. qHqxsicktQHr CNWldugL bXluvRhlEmfs",
    "...": "..."
}

11. Bulk Onboard Staff
Onboards multiple staff members from an uploaded spreadsheet file.

Endpoint: POST /api/v1/staff/bulk-onboard

Request Body:

This endpoint expects multipart/form-data. You must upload a file (e.g., .xlsx, .csv) with the field name staffFile.

Expected Output (200 OK):

A summary of the bulk import process, including successes and failures.

JSON

{
  "totalRows": 50,
  "successfulCount": 48,
  "errors": [
    {
      "row": 15,
      "error": "Invalid CollegeMasterID"
    },
    {
      "row": 32,
      "error": "Duplicate PersonalEmail"
    }
  ]
}
